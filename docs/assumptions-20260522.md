## 補足事項

### 環境選択（前提6）

- ローカル Mac: Docker Desktop + kind で手軽に起動可能
- PVE Ubuntu: リソースが潤沢な場合は kind をリモート操作してもよい
- どちらの環境でも `infra/kind-config.yaml` はそのまま使用可能

### ディレクトリ構造の柔軟性（前提5）

要件定義では component1〜4 の固定構造を定義しているが、  
コードリーディングを進める中でより適切な分割方法が見つかれば変更してよい。  
変更時はこのファイルと `docs/requirements.md` も更新すること。
